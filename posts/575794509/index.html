1. GNN Augmentation

  • 图特征增强

  • 图结构增强

1.1 Why Augment Graphs？

Our assumpution so far: Raw input graph = computational graph

Reasons for breaking this assumption:

  • Features:
    • The input graph lacks features
  • Graph structure:
    • The graph is too sparse: inefficient message passing
    • The graph is too dense: message passing is too costly
    • The graph is too large: cannot fit the computational graph into a GPU

It’s unlikely that the input graph happens to be the optimal computation graph for embeddings

Augmentation Approaches

  • Features:

    • The input graph lacks features $\rightarrow$ feature augmentation

  • Graph structure:

    • The graph is too sparse: $\rightarrow$ Add virtual nodes/edges
    • The graph is too dense: $\rightarrow$ Sample neighbors when doing message passing
    • The graph is too large: $\rightarrow$ Sample subgraphs to compute embeddings
      • will cover in Scaling up GNNs

1.2 Feature Augmentation

Why do we need feature augmentation?

1.2.1 输入图不含结点特征

例如输入只是图邻接矩阵

标准方法：

  • 为结点分配一个常值
  • 为结点分配唯一的 ID，它们会被转换为 one-hot 向量
Constant node feature One-hot node feature
Expressive power Medium. All the nodes are identical, but GNN can still learn from the graph structure High. Each node has a unique ID, so node-specific information can be stored
Inductive learning (Generalize to unseen nodes) High. Simple to generalize to new nodes: we assign constant feature to them, then apply our GNN Low. Connot generalize to new nodes: new nodes introduce new IDs, GNN doesn’t know how to embed unseen IDs
Computational cost Low. Only 1 dimensional feature High. $O\left(V\right)$ dimensional feature, cannot apply to large graphs
Use cases Any graph, inductive settings (generalize to new nodes) Small graph, transductive settings (no new nodes)
1.2.2 某些结构对于GNN很难学习

  • 所有图中结点度数为2
  • 计算图为相同的二叉树

解决方法：

使用循环计数信息来增加结点特征

其他常用的增强特征的方法：

  • Node degree
  • Clustering coefficient
  • PageRank
  • Centrality

1.3 The graph is too sparse: $\rightarrow$ Add virtual nodes/edges

1.3.1 增加虚拟边
  • Connect 2-hop neighbors via virtual edges

  • 邻接矩阵 $A\rightarrow A+A^2$

  • Use cases: Bipartite graphs

1.3.2 增加虚拟结点
  • 虚拟结点连接图中所有点
  • 不需要很深的神经网络用于通信
  • 所有结点之间有：Node A – Virtual – Node B
  • Benefits: Greatly improves message passing in sparse graphs

1.4 The graph is too dense: $\rightarrow$ Node Neighborhood Sampling

Idea: (Randomly) sample a node’s neighborhood for message passing

尽管可以随机，但有更好的启发式方法

  • 随机选择两个邻居来传递给定层的消息
  • 在下一次训练，选取不同的邻居来传递消息
  • 极大降低计算成本

2. Training Graph Neural Networks

2.1 Different prediction heads

  • Node-level tasks
  • Edge-level tasks
  • Graph-level tasks
2.1.1 Node-level prediction

直接使用结点嵌入进行预测，在 GNN 计算之后，我们有 $d$ 维嵌入：$\left\{h_v^{(l)}\in \mathbb{R}^d,\forall v\in G\right\}$

如果要进行 $k$ 路预测，比如 $k$ 类的分类或者 $k$ 个目标的回归

$W^{(H)}$ 将嵌入从 $h_v^{(L)}\in \mathbb{R}^{k\times d}$ 映射到 $\hat{y}_v\in \mathbb{R}^k$ 用于计算损失

2.1.2 Edge-level prediction

使用结点嵌入对进行预测，$k$ 路预测

Options for $Head_{edge}(h_u^{(L)},h_v^{(L)})$

  1. Concatenation + Linear
  • $\hat{y}_{uv}=Linear(Concat(h_u^{(L)},h_v^{(L)}))$
  • Linear$(\cdot)$ 将 $2d$ 维嵌入映射到 $k$ 维嵌入
  1. Dot product
  • $\hat{y}_{uv}=(h_u^{(L)})^Th_v^{(L)}$
  • 只能用于 1 路预测，比如边的存在性
  • 推广到 $k$ 路预测，与 Multi-head attention 类似，可供训练的参数 $W^{(1)},\dots,W^{(k)}$
2.1.3 Graph-level prediction

使用所有结点嵌入做预测，$k$ 路预测

$Head_{graph}(\cdot)$ 与 GNN 层中的 AGG$(\cdot)$ 类似

Options for $Head_{graph}\left(\left\{h_v^{(L)}\in \mathbb{R}^d,\forall v\in G\right\}\right)$

  1. Global mean pooling
  2. Global max pooling
  3. Global sum pooling

这些选择在较小的图上表现更好，对于较大的图呢？

在较大图上全局池化会损失大量信息

Toy example:

  • Node embeddings for $G_1$: {-1, -2, 0, 1, 2}
  • Node embeddings for $G_2$: {-10, -20, 0, 10, 20}
  • Clearly $G_1$ and $G_2$ have very different node embeddings $\rightarrow$ Their structures should be different
  • If we do global sum pooling
  • prediction for $G_1$: $\hat{y}_G=Sum(\left\{-1, -2, 0, 1,2\right\})=0$
  • prediction for $G_2$: $\hat{y}_G=Sum(\left\{-10, -20, 0, 10,20\right\})=0$
  • We cannot differentiate them

Hierarchical Global Pooling

  • We will aggregate via ReLU(Sum$(\cdot)$)
    • We first separately aggregate the first 2 nodes and last 3 nodes
    • Then we aggregate again to make the fianl prediction
  • $G_1$: {-1, -2, 0, 1, 2}
    • $\hat{y}_a=ReLU(Sum(\left\{-1, -2\right\}{}))=0,\hat{y}_b=ReLU(Sum(\left\{0,1, 2\right\}{}))=3$
    • $\hat{y}_G=ReLU(\left\{\hat y_a, \hat y_b\right\})=3$
  • $G_2$: {-10, -20, 0, 10, 20}
    • $\hat{y}_a=ReLU(Sum(\left\{-10, -20\right\}{}))=0,\hat{y}_b=ReLU(Sum(\left\{0,10, 20\right\}{}))=30$
    • $\hat{y}_G=ReLU(\left\{\hat y_a, \hat y_b\right\})=30$

Hierarchical Pooling In Pracice

DiffPool idea

  • Hierarchically pool node embeddings
  • Leverage 2 independent GNNs at each level
    • GNN A: Compute node embeddings
    • GNN B: Compute the cluster that a node belongs to
  • GNNs A and B at each level can be executed in parallel

2.2 Predictions and Labels

  • Supervised labels
    • Labels come from external sources
  • Unsupervised signals
    • Signals come from graph themselves
  • Somtimes the differences are blurry
2.2.1 Supervised Labels on Graphs

监督标签来自特定的用例

  • Node labels $y_v$: 在引文网络中，一个结点属于哪个主题区域
  • Edge labels $y_{uv}$: 在交易网络中，哪些边（交易）是欺诈的
  • Graph labels $y_G$: 在分子图中，图的药物相似性

Advice: Reduce your task to node / edge / graph labels, since they are easy to work with, E.g., we knew some nodes form a cluster. We can treat the cluster that a node belongs to as a node label

2.2.2 Unsupervised Signals on Graphs

有时候我们只有图，没有任何的外部标签。

The solution: 定义自监督学习任务，监督信息来自输入图本身

let GNN predict the following:

  • Node-level $y_v$: Node statistics: such as clustering coefficient, PageRank
  • Edge-level $y_{uv}$: Link prediction: hide the edge between two nodes, predict if there should be a link
  • Graph-level $y_G$: Graph statistics: for example, predict if two graphs are isomorphic4

2.3 Loss function

  • Classification loss
  • Regression loss

We have $N$ data points

  • Each data point can be a node / edge / graph
  • Node-level: prediction $\hat{y}_v^{(i)}$, label $y_v^{(i)}$
  • Edge-level: prediction $\hat{y}_{uv}^{(i)}$, label $y_{uv}^{(i)}$
  • Graph-level: prediction $\hat{y}_G^{(i)}$, label $y_G^{(i)}$

统一记作 prediction $\hat{y}^{(i)}$, label $y^{(i)}$

Classification or Regression

Classification: $y^{(i)}$ 为离散的分类值

Regression: $y^{(i)}$ 为连续值

Differences: loss function & evaluation metrics

  • Classification Loss use cross entropy
  • Regression Loss use Mean Squared Error (MSE)

3. Dataset

3.1 Dataset Split: Fixed / Random Split

  • Fixed split
    • Training set
    • Validation set
    • Test set
  • Random split: we will randomly split out dataset into training / validation/ test
    • We report average performance over different random seeds

3.2 Why Splitting Graphs is Special

  • Image classification: Each data point is an image

    • Here data points are independent

  • Node classification: Each data point is a node

    • Here data points are NOT independent

3.3 Solution

3.3.1 Transductive setting:

The input graph can be observed in all the dataset splits (training, validation and test set)

We will only split the (node) label

  • At training time, we compute embeddings using the entire graph, and train using node 1&2’s labels
  • At validation time, we compute embeddings using the entire graph, and evaluate on node 3&4’s labels
3.3.2 Inductive setting:

We break the edges between splits to get multiple graphs

  • At training time, we compute embeddings using the graph over node 1&2, and train using node 1&2’s labels
  • At validation time, we compute embeddings using the graph over node 3&4, and evaluate on node 3&4’s labels
Transductive setting Inductive setting
on the same graph on different graph
The dataset consists of one graph The dataset consists of multiple graphs
The entire graph can be observed in all dataset splits, we only split the labels Each split can only observe the graph within the split. A successful model should generalize to unseen graphs
Only applicable to node / edge prediction tasks Applicable to node / edge / graph tasks

3.4 Graph Classification

Only the inductive setting is well defined for graph classification

这里有多个图，将它们分别放入训练、验证和测试集

3.5 Link Prediction

predict missing edges

  • Setting up link prediction is tricky
  • Link prediction is unsupervised / self-supervised task. We need to create the labels and dataset splits on our own
  • Concretely, we need to hide some edges from GNN and let the GNN predict if the edges exist

For link prediction, we will split edges twice

  • Step 1: Assign 2 types of edges in the original graph
    • Message edges: Used for GNN message passing
    • Supervision edges: Used for computing objectives
    • After step 1:
      • Only message edges will remain in the graph
      • Supervision edges are used as supervision for edge predictions made by the model, will not be fed into GNN!
  • Step 2: Split edges into train / validation / test
  • Option 1: Inductive link prediction split
    • Suppose we have a dataset of 3 graphs. Each inductive split will contain an independent graph
    • In train or val or test set, each graph will have 2 type of edges: message edges + supervision edges
      • Supervision edges are not the input to GNN
  • Option 2: Transductive link prediction split
    • This is the default setting when people talk about link prediction
    • Suppose we have a dataset of 1 graph
    • By defination of “transductive”, the entire graph can be observed in all dataset splits
    • But since edges are both part of graph structure and the supervision, we need to hold out validation / test edges
    • To train the training set, we further need to hold out supervision edges for the training set

Summary: Transductive link prediction split:

  • Training message edges
  • Training supervision edges
  • Validation edges
  • Test edges

Luckily, we have full support in DeepSNAP and GraphGym
